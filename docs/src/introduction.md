# Introduction

**openbabel-rs** provides safe, idiomatic Rust bindings to the
[OpenBabel](https://openbabel.org) cheminformatics toolkit. It lets you read and
write molecules in dozens of chemical file formats, compute properties and
descriptors, match SMARTS patterns, generate 2D/3D structures, run force-field
energy minimizations, and much more — all from Rust, without writing any C++.

Under the hood, the crate links OpenBabel's C++ library through a thin shim
built with [`cxx`](https://cxx.rs). OpenBabel itself is compiled from source as
part of the build, so there is no system dependency on a pre-installed OpenBabel.

```rust,no_run
use openbabel::Molecule;

let mut mol = Molecule::parse("CCO", "smi")?;   // ethanol, from SMILES
assert_eq!(mol.formula(), "C2H6O");
mol.add_hydrogens();
println!("{}", mol.write("can")?);              // canonical SMILES
# Ok::<(), openbabel::Error>(())
```

## Workspace layout

The repository is a Cargo workspace of three crates:

| Crate           | Role                                                                 |
| --------------- | ------------------------------------------------------------------- |
| `openbabel-sys` | Low-level FFI: a `cxx` bridge plus a C++ shim. Its `build.rs` compiles OpenBabel from source and links it. |
| `openbabel`     | The safe, idiomatic API: `Molecule`, `Atom`, `Bond`, `Residue`, `SmartsPattern`, `Fingerprint`, `Transform`, `Minimizer`, the `elements` module, `Error`, … |
| `cli`           | `openbabel-demo` — a small SMILES-inspection demo binary.           |

Dependencies are vendored as git submodules: OpenBabel at `vendor/openbabel-src`
(tag `openbabel-3-2-1`) and Eigen (header-only, used for structure alignment and
distance-geometry) at `vendor/eigen` (tag `3.4.0`).

## Scope at a glance

The safe API currently covers: molecule I/O in any OpenBabel format; basic
properties (formula, mass, charge, counts); atom/bond access; SMARTS
substructure matching; fingerprints and Tanimoto similarity; numeric
descriptors (logP, TPSA, …); 2D depiction to SVG; 3D coordinate generation;
force-field single-point energy and constrained geometry optimization; partial
charges; InChI/InChIKey identifiers; least-squares structure alignment;
stereochemistry perception; SMARTS-based structural transforms; conformer
search; periodic-table element data; residues (biopolymer/PDB substructure); and
spectral descriptors. See the [Feature Tour](./feature-tour.md) for details.
